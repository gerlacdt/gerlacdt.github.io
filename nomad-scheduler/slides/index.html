Nomad

A distributed, high available, datacenter-aware cluster manager/scheduler

Daniel Gerlach@2016

Nomad reference architecture

Advantages

  • seamless consul integration
    • no interfering health checks: ecs / elb vs. consul
    • zero-downtime deployments
    • gliderlabs/registrator obsolete --> nomad (de)registers services
  • dynamic ports --> better resource utilization
  • service accessible outside of consul cluster via fabio

More Advantages

  • powerful CLI
    • direct log access
  • declarative job definition under version control
  • periodic batch jobs
    • distributed cron
    • utilize multiple machines
  • easy setup (single golang-binary)
  • docker support and more (raw executables, jvm)

Deployments

  • canary deployment
  • blue-green deployment
  • fine-grained route weighting
    • 10% experimental, 90% stable
  • deployment strategy configurable
    • step-by-step rollout
    • how many instances in parallel

Disadvantages

  • it's not kubernetes (all-in-one-package)
  • relatively new --> version 0.4.1
  • self-hosted
